# 博客全站视觉升级设计

## 目标

将现有轻量 Jekyll 博客升级为适合长期维护的中文个人技术博客。首页以最新文章为核心，文章页以舒适阅读为核心，全站具备一致的视觉系统、响应式布局和深色模式。

## 视觉方向

- 风格：极简阅读 / 个人主页
- 语言：中文为主
- 色彩：黑白灰为主，暖橙色作为克制的强调色
- 排版：清晰、宽松、内容优先；中文使用系统无衬线字体，标题更有力度
- 动效：只使用简短的悬停、主题切换和移动导航动效，并尊重 `prefers-reduced-motion`

## 信息架构

### 全局导航

左侧显示站点名称，右侧提供首页、文章、关于、GitHub 和主题切换。移动端保留核心入口，通过按钮展开导航。

### 首页

首页采用“置顶首篇 + 文章流”：

1. 顶部短介绍说明作者和博客主题。
2. 最新文章以更强层级展示标题、摘要、日期和阅读时长。
3. 其余文章使用紧凑列表，便于快速浏览。

### 文章页

文章页使用约 `680px` 的阅读宽度，展示标题、发布日期、阅读时长和正文。正文统一优化标题、列表、链接、引用、代码块、表格和图片。

### 页脚

页脚提供版权信息、GitHub 链接和返回顶部入口。

## 技术方案

- 保持 GitHub Pages 原生支持的 Jekyll 架构，不引入框架或构建依赖。
- 使用 `_layouts/default.html` 作为统一站点外壳。
- 新增 `_layouts/post.html` 处理文章元信息和正文语义。
- 使用 `stylesheets/stylesheet.css` 提供设计令牌、布局、组件、深色模式与响应式规则。
- 使用 `javascripts/main.js` 管理深色模式、移动导航和当前年份。
- 使用 Jekyll front matter 的 `description` 字段作为文章摘要，缺失时回退到正文截断。

## 可访问性与性能

- 使用语义化元素、跳转到正文链接、可见键盘焦点和按钮 `aria` 属性。
- 默认跟随系统主题，用户手动选择后通过 `localStorage` 保存。
- 不加载外部字体或 UI 库，避免额外网络请求。
- 控制正文行宽和图片尺寸，移动端无横向滚动。
- 主题与导航脚本在无 JavaScript 时仍保留可读内容和核心导航。

## 验收标准

- 首页清晰展示最新文章与其余文章列表。
- 文章详情页具备完整、舒适的中文阅读排版。
- 桌面与移动端均可正常浏览，无水平溢出。
- 浅色、深色和系统主题均可正常工作，手动选择可持久化。
- 键盘可访问导航、主题按钮和返回顶部链接。
- GitHub Pages/Jekyll 可正常构建。
